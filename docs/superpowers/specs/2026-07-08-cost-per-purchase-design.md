# 구매당비용·개당마진 컬럼 추가 — 설계

작성일: 2026-07-08

## 목적

쿠팡 마진표의 **개당마진**은 곧 손익분기점이다. 상품 1개를 팔 때 광고비를 개당마진만큼 쓰면 순이익이 0이 된다.
현재 결과표에는 총합(마진(광고전)=개당마진×수량, 사용광고비 총액)만 보여, "1개당 실제로 쓴 광고비"가 드러나지 않는다.
판매수량 오른쪽에 **구매당비용**과 **개당마진**을 나란히 두어, 상품별 손익분기 여부를 한눈에 보이게 한다.

핵심 항등식: `순이익 = 판매수량 × (개당마진 − 구매당비용)` → 구매당비용이 개당마진을 넘으면 그 상품은 적자.

## 추가 컬럼 (상품별 상세에만)

판매수량 바로 오른쪽에 2개 삽입. 새 상세 컬럼 순서:

```
날짜 · 상품구분 · 상품명 · 등록상품ID · 판매수량 · [구매당비용] · [개당마진] ·
인사이트매출 · 정산매출 · 사용광고비 · 마진(광고전) · 순이익 · 순이익률
```

- **구매당비용** = `사용광고비 ÷ 판매수량` (반올림). 판매수량 0이면 `-` (파생값, 저장 안 함)
- **개당마진** = 마진표의 개당마진 (`= 마진(광고전) ÷ 판매수량`). 손익분기 기준선. 엑셀에 실제 저장되는 컬럼.

## 색상 규칙 (구매당비용 셀, 웹 표)

- 구매당비용 < 개당마진 → `pos`(초록, 흑자구간)
- 구매당비용 ≥ 개당마진 → `neg`(빨강, 적자구간)
- 판매수량 0 → `muted`("-")

## 합계행

- 구매당비용 = `Σ사용광고비 ÷ Σ판매수량` (블렌디드)
- 개당마진 = `Σ마진(광고전) ÷ Σ판매수량` (가중평균)
- 두 값도 색상 규칙 동일 적용.

## 적용 위치 (상세만, 요약은 그대로)

- 웹 결과표 (`renderResults`)
- 엑셀 '상세' 시트 (`downloadExcel`) — 헤더 메모·컬럼폭·숫자서식 포함
- 웹 범례 (`renderLegend`) 및 툴팁 (`COLUMN_DEFS`)
- 구글시트 '상세' 탭 (`detailRow` + `apps-script.gs`)
- **일별요약·구글시트 요약 탭은 변경 없음**

## 재업로드(왕복) 호환

- 개당마진은 저장 컬럼이므로 `parseResult`에서 그대로 읽음.
- 옛 결과 엑셀(컬럼 없음)은 `마진(광고전) ÷ 판매수량`으로 복원 → 기존 누적 엑셀도 무손실 동작.
- 구매당비용은 파생값이라 별도 저장·복원 불필요.

## 구글시트 재배포 (사용 시에만)

구글시트 '상세' 탭 컬럼이 늘어나므로:
1. `apps-script.gs`의 `DETAIL_HEADER`, `formatSheet_` 인덱스 갱신본으로 교체
2. Apps Script [배포]>[배포 관리]에서 기존 배포 수정(새 버전)
3. 기존 '상세' 탭은 옛 헤더(11열)를 그대로 갖고 있으므로, **'상세' 탭을 삭제**한 뒤 앱에서 '전체 전송'으로 재생성(누적본은 localStorage에 있으므로 무손실)

구글시트를 쓰지 않으면 위 단계는 무관.
